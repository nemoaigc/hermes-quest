页面布局方案

NPC 对话的位置 — 三种方案对比

A: NPC 独立 Tab

4 个 tab: MAP / GUILD / SHOP / NPC。NPC 页面里可以选择不同 NPC 对话，有完整的对话界面。

Pros

  • 对话空间大，多 NPC 可展开
  • 不挤占其他功能

Cons

  • 4 个 tab 略多
  • 对话和公会分开，逻辑上 NPC 应该在公会里

B: 中间底部对话栏

3 个 tab: MAP / GUILD / SHOP。对话框固定在中间区域底部，像 RPG 的对话框一样始终可见。上面是内容，下面是 NPC 对话。

Pros

  • 最像 RPG 对话体验
  • 对话始终可用，不用切 tab
  • 和 DialogBox 组件风格一致

Cons

  • 地图/公会内容区域变小
  • 对话框高度有限

C: Guild 内整合 NPC

3 个 tab: MAP / GUILD / SHOP。Guild 页面上半部分是公告板+任务，下半部分是 NPC 选择 + 对话。多个 NPC 头像可点击切换。

Pros

  • NPC 在公会里最合逻辑
  • 公告板和对话联动自然（NPC 推荐任务 → 公告板更新）
  • 多 NPC 头像选择很有 RPG 感
  • 只要 3 个 tab，不臃肿

Cons

  • Guild 页面内容较多
